# API reference

This section points to the raw OpenAPI spec for the full endpoint surface.

## Source of truth

- `docs/openapi.yaml`

## Implementation guidance

Use the OpenAPI spec for:

- request and response schemas
- status codes
- auth headers
- idempotency keys
- webhook payload shapes

Do not duplicate endpoint details in prose unless the prose explains behavior that is not obvious from the schema.
