# Integration guide

This is the partner-facing path from zero to a working settlement integration.

## Recommended order

1. Read the introduction and mechanics pages.
2. Request API credentials.
3. Create a test order.
4. Confirm webhook receipt.
5. Validate local currency payout state.
6. Move to production only after you can reconcile every state transition.

## Minimum implementation

- Store your order IDs
- Store your webhook cursor
- Store the selected corridor
- Reconcile order state on every callback
- Retry safely when an order is not in a terminal state

## What not to do

- Do not hard-code settlement assumptions.
- Do not assume every corridor has the same payout timing.
- Do not treat “request accepted” as “settled.”
