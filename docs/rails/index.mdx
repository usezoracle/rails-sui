# Introduction

Rails is a Sui-native settlement infrastructure for converting native SUI, USDC, and USDT into local fiat in emerging markets. It is built for fintechs, consumer apps, DeFi protocols, and cross-border payment systems that need a clear custody boundary and predictable settlement behavior.

Rails supports EVM integration, but settlement happens on Sui. The system uses on-chain escrow plus off-chain liquidity and payout rails to move value from stablecoins into local currencies without Rails custodying user funds.

## What Rails is for

- Stablecoin-to-fiat settlement
- Merchant and partner payouts
- API-driven payment workflows
- Product systems that need an auditable settlement core

## What Rails is not for

- General-purpose wallet hosting
- Consumer trading UI
- Arbitrary on-chain token swapping
- Custody of user balances outside the settlement flow

## Core supported corridors

Rails currently supports local payout flows for:

- KES
- NGN
- TZS
- MWK

## Core supported assets

- Native SUI
- USDC
- USDT
