# Getting started

This guide covers the smallest useful integration path: authenticate, create a quote, create an order, and listen for settlement updates.

## Prerequisites

- A Rails environment and API credentials
- A backend that can store your order IDs
- A webhook endpoint or polling loop
- A currency and asset pair that Rails supports

## First integration steps

1. Choose the asset you want to receive, usually `SUI`, `USDC`, or `USDT`.
2. Choose the local fiat corridor you want to pay out, such as `KES`, `NGN`, `TZS`, or `MWK`.
3. Request a quote.
4. Create an order from that quote.
5. Watch for status changes until settlement completes.

## Implementation rule

Keep the order lifecycle in your own database.

Rails should be the system of record for settlement state, but your app should persist:

- the order reference
- the user reference
- the quoted amount
- the selected corridor
- the current state

## Test versus production

Use one environment for integration testing and one for live value movement. Do not mix them.
