Rails is a decentralized, non-custodial settlement protocol built on the **Sui blockchain**. It enables instant, peer-to-peer conversion of stablecoins (such as USDC and USDT) into local fiat currencies (e.g., NGN, KES, TZS, MWK) in emerging markets through a decentralized Liquidity Provider (LP) settlement system.

Designed for fintechs, consumer apps, DeFi protocols, and cross-border payment networks, Rails coordinates the transaction lifecycle from on-chain escrow to off-chain fiat payout without taking long-term custody of user funds.

---

## 💡 Core Concepts & How It Works

Rails ensures secure, atomic settlement using a hybrid model:
1. **On-Chain Escrow:** When a payment is initiated, funds are locked in a shared Sui Move object (`Order<T>`) governed by the [contracts/gateway/](file:///Users/mac/rails/contracts/gateway/) package.
2. **Off-Chain Fiat Payout:** A decentralized Liquidity Provider (LP) picks up the transaction off-chain and executes the payout in local currency (e.g., via bank transfer or mobile money).
3. **Escrow Release:** Once the fiat payout is confirmed (via BaaS webhooks or provider proofs), the on-chain gateway releases the escrowed stablecoins to the LP.
4. **Tapp Cards:** Physical cards powered by NFC technology. Users pre-fund a `CardSpendingCap<T>` on-chain. When tapped, the card initiates a secure debit call via the off-chain aggregator within Move-enforced daily and per-tap limits.

## 🚀 Getting Started

### Prerequisites
- **Go:** 1.25+
- **Sui CLI:** Required for compiling/testing Move packages.
- **PostgreSQL:** 13+ (for database storage)
- **Redis:** Required for rate-limiting, cron state, and keyspace subscriptions.
- **Air** (Optional): Hot-reload daemon for Go development.

---

### Installation & Local Setup

#### 1. Clone the Repository
```bash
git clone git@github.com:usezoracle/rails-sui.git
cd rails-sui
```

#### 2. Configure Environment Variables
Copy the example environment file and configure the settings (ports, databases, API credentials, and Sui network configurations):
```bash
cp .env.example .env
# Edit .env with your local PostgreSQL, Redis, and Sui configurations
```

#### 3. Database Schema Generation
Generate the Ent ORM client and run the database schema migrations:
```bash
# Generate Ent schemas
make gen-ent

# Ensure the PostgreSQL database is created
createdb rails
```
*Note: In the `local` environment, schema migrations run automatically on startup.*

#### 4. Spin up Redis
Ensure Redis is running locally:
```bash
# macOS
brew services start redis
```

#### 5. Build and Test Move Contracts
Navigate to the Sui contracts directory and compile/test the Move package:
```bash
cd contracts/gateway
sui move build
sui move test
cd ../..
```

#### 6. Run the Go Backend
Start the backend server in development mode using hot reloading:
```bash
# Using Makefile helper (starts backend + runs Ent generation)
make run

# Alternatively, run directly
go run main.go
```
The server will boot and begin listening on `http://localhost:8000` (or `SERVER_PORT` defined in your `.env`).

## 🧪 Testing

To run the complete test suite for the Go services and Move smart contracts, execute:

```bash
# Run all Go tests
make test

# Run Go coverage analysis
make test-coverage

# Run Sui Move contract tests
cd contracts/gateway && sui move test
```
